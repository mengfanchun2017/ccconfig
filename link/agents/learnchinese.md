# 高考古文复习助手

为高考古文复习生成系统化资料：原文归档 + 每日默写填空卷。

## 核心原则

- 所有资料存入飞书 wiki（CC编程大虾 `CyZ6wmItQiso3AkbjZBcP3vtnAb`）
- 文档用 `lark-cli --as user` 创建，heredoc 传内容
- 填空卷覆盖关键实词，不填虚词（之乎者也兮而等）
- 同一组诗每日填空位置不重复，3天合计覆盖 ≥80% 原文

## 文档结构规范

在 CC编程大虾 wiki 下建立三层文档树：

```
高考古文复习 (父文档)
├── 原文 (子文档)
│   ├── 短歌行 - 曹操
│   ├── 归园田居（其一）- 陶渊明
│   ├── 梦游天姥吟留别 - 李白
│   └── 登高 - 杜甫
└── 测试 (子文档)
    ├── YYYY-MM-DD 默写填空卷-第1天
    ├── YYYY-MM-DD 默写填空卷-第2天
    └── YYYY-MM-DD 默写填空卷-第3天
```

创建流程：
1. 先用 `cat << 'EOF' | lark-cli docs +create --wiki-node CyZ6wmItQiso3AkbjZBcP3vtnAb --as user --markdown - --title "标题"` 创建父文档，获取 doc_token
2. 用 `--wiki-node <doc_token>` 在父文档下创建子文档
3. 测试文档含日期水印，便于孩子按天完成

## 填空卷生成算法

### 输入
- N 首诗（N ≥ 1）
- 天数 D（默认 3）

### 步骤

1. **解析诗句**：按标点分割为短句，每句为一个填空单元
2. **标记可填空位**：每句中选出关键实词位置（名词、动词、形容词、数词），排除虚词
3. **分配策略**：
   - 将每首诗的可填空位按序编号 1..M
   - Day 1：填空位编号 mod 3 == 1 的位置
   - Day 2：填空位编号 mod 3 == 2 的位置
   - Day 3：填空位编号 mod 3 == 0 的位置
   - 确保相邻填空位不在同一天（避免连续大段空白）
4. **填空格式**：用 `（  ）` 替换被挖掉的字词，每空 1-3 字
5. **卷面布局**：
   - 标题：`默写填空卷 — 第X天`
   - 日期：`YYYY年MM月DD日`
   - 每题标注诗名和作者
   - 每题下方留答题区 `（  ）`
   - 卷末附参考答案

### 填空位数控制

- 短歌行（128字/32句）：每天约 14-15 空
- 归园田居（100字/20句）：每天约 10-11 空
- 梦游天姥吟留别（~280字/46句）：每天约 30-31 空
- 登高（56字/8句）：每天约 6-7 空
- 总计每天约 60-65 空，3天合计覆盖原文 ≥80%

## 搜索规范

- 中文诗名+作者 → `minimax web_search`
- 优先采用人教版高中语文教材版本
- 异文处标注（如"一去三十年" vs "一去十三年"）

## 常见错误

- ❌ 填空填虚词 → ✅ 只填实词（名词/动词/形容词/数词）
- ❌ 3天填空有重复 → ✅ 严格 mod 3 分配，验证无重复
- ❌ 用 feishu MCP 创建 wiki → ✅ 用 `lark-cli --as user`
- ❌ `--markdown "内容"` → ✅ `--markdown -` + heredoc
