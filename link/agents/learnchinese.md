# 高考古文复习助手

为高考古文复习生成系统化资料：原文归档 + 每日默写填空卷。

## 核心原则

- 所有资料存入飞书 wiki（CC编程大虾 `CyZ6wmItQiso3AkbjZBcP3vtnAb`）
- 文档用 `lark-cli --as user` 创建，heredoc 传内容
- 填空卷以半句为单位（逗号分割），混用三种挖空方式
- 全角下划线 `＿` (U+FF3F) 表示空白，避免 markdown 解析，方便打印
- 横线长度 = 原文字数 × 1.2（留20%余量）
- 4天为一轮，每天含填空+理解性默写，平均分配

## 文档结构规范

在 CC编程大虾 wiki 下建立三层文档树：

```
高考古文复习 (父文档)
├── 原文 (子文档)
│   ├── 短歌行 - 曹操
│   ├── 归园田居（其一）- 陶渊明
│   ├── 梦游天姥吟留别 - 李白
│   └── 登高 - 杜甫
└── 测试 (子文档)
    ├── 第1天（填空+理解性默写）
    ├── 第2天（填空+理解性默写）
    ├── 第3天（填空+理解性默写）
    ├── 第4天（填空+综合复习）
    └── 理解性默写答案汇总
```

创建流程：
1. 先用 `cat << 'EOF' | lark-cli docs +create --wiki-node CyZ6wmItQiso3AkbjZBcP3vtnAb --as user --markdown - --title "标题"` 创建父文档，获取 doc_token
2. 用 `--wiki-node <doc_token>` 在父文档下创建子文档
3. Python 脚本 `/tmp/gen_poetry.py` 生成每日内容

## 填空卷生成算法（v3）

### 核心原则
- 以逗号分割每句诗为前后两半，52句全部参与
- 三种填空方式轮换（每天按 `(句子编号 + 天偏移*3) % 4` 分配）：
  - **Type A** 前半句挖空：`＿＿＿，后半句` — 有后半提示
  - **Type B** 后半句挖空：`前半句，＿＿＿` — 有前半提示
  - **Type C** 上下文默写：`前一句全文 → ＿＿＿，＿＿＿` — 训练前后关联
- 4天每句至少出现一次，每种填空方式都用到
- 全角下划线 `＿` (U+FF3F) 长度 = ceil(原文字数 × 1.2)，最少4个

### 每日结构
- 填空部分：52句全部出现，Type A/B/C混用
- 理解性默写：3题，从12题题库轮换（每天不重复）
- 含参考答案

### 理解性默写题库
12题覆盖4首诗（每首3题），含开放式题目，贴近高考6分情境式命题。

## 填空卷文档格式

```markdown
# 第X天

## 一、默写填空

> 横线长度已留出约20%余量，可直接书写。

### 短歌行 — 曹操

1. ＿＿＿＿＿，人生几何！
2. 譬如朝露，＿＿＿＿＿＿
3. 譬如朝露，去日苦多。→ ＿＿＿＿＿，＿＿＿＿＿＿
...

## 二、理解性默写

1. 《短歌行》中，曹操以"周公吐哺"的典故自比，表达求贤若渴之心的两句是：
   ______，______。

**参考答案**
1. 周公吐哺，天下归心。
```

## 搜索规范

- 中文诗名+作者 → `minimax web_search`
- 优先采用人教版高中语文教材版本
- 异文处标注（如"一去三十年" vs "一去十三年"）

## 常见错误

- ❌ 填空只填实词 → ✅ 默写半句/整句（贴近高考）
- ❌ 用半角 `_` 下划线 → ✅ 用全角 `＿` (U+FF3F)，避免 markdown 解析
- ❌ 题型单一 → ✅ 三种填空方式混用 + 理解性默写
- ❌ 用 feishu MCP 创建 wiki → ✅ 用 `lark-cli --as user`
- ❌ `--markdown "内容"` → ✅ `--markdown -` + heredoc
