# 报告生成规范（全局约束）

> 飞书报告生成的格式、结构、内容硬约束。完整工作流 → `skills/f-report-gen/SKILL.md`。

## 格式
- **标题层级**：H1 为默认顶级标题（非 H2），每个 H1 下设一段简短概括说明（2-3句）
- **小结章节**：如"关键结论""总结"等也使用 H1，且同样加概括说明
- **编号**：不加手动编号（飞书自动生成目录）
- **风格**：专业正式的研究报告体，体现分析逻辑，清晰说明结果与原因

## 数据呈现
- **对比场景**（如自有部署 vs MaaS）：优先使用表格或结构化图表，避免纯列表罗列
- **多层级架构图**：每层如有多个组件，横向并列展示，避免全纵向过长
- **数据标准引用**：实事求是，不简化国标内容。如国标为三级但场景仅涉及两级，应完整说明国标三级体系，再注明场景适用范围

## 内容约束
- **不写"PPT还原与分析"**：PPT 以附件形式保存，研究报告聚焦分析本身
- **通用性/不写具体企业名**：报告内容保持通用性，使用"该企业""目标场景""该航司"等泛指表述，不写具体企业名称

## 数值与单位
- 成本数据使用实际量级（万元级非千元级），与真实用量匹配

## 末尾参考节（必）
- 报告末尾必加 "## 参考" H1 章节，**不计入**目录正文（不进入 H1 序列）
- 按类型分组：数据源 / 模板 / 工具 / 引用标准
- 每条 1 行：`[显示文本](url) — 一句话用途`，链接不能裸 URL
- 描述中按需内联引用（如"基于 X 报告""使用 Y 模板"），不堆末尾
- 来源追溯清单（如 f-research 三源搜索清单）放在"数据源"组下
