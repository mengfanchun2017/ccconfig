# 编码规范

## 禁止的操作
- rm -rf, git reset --hard, git clean -f, mkfs, dd, chmod -R 777
- sudo（除 apt-get 外）
- 重定向覆盖 (:/>)

## 代码风格
- 默认不写注释，只在 WHY 非显然时加一行
- 不添加错误处理/fallback/验证给不可能发生的场景
- 不设计假想的未来需求，三个相似行优于过早抽象
- 不写多行 docstring 或注释块
- 优先编辑现有文件，不新建除非必须
- 不引入安全漏洞（SQL 注入、XSS、命令注入等）

## 终端输出
- 避免用表格格式展示 URL 或单条信息。表格单元格内长链接会被截断换行，WSL Windows 终端中用户需手动选中复制两行才能拼接完整 URL
- 文档/链接创建成功后，直接输出 markdown 链接格式 `[标题](URL)`，链接独占一行方便点击

## CLI 试错自更新
- 任何 CLI 命令的 flag/参数通过试错发现正确用法后，立即追加到对应的速查 rules（如 `feishu-cli-cheatsheet.md`）
- 无对应速查文件时新建，保持简短：命令→参数对照表 + 错误→正确映射
- 目标：同一错误不犯第二次
