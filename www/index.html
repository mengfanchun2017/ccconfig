FAQ

你可能会问。

和 dotfiles 有什么不同？
Dotfiles 同步文件。ccconfig 管理版本、依赖、自动更新，并用符号链接桥接公开/私密仓库。它是带构建系统的 dotfiles — setup-links.sh 理解完整的 ~/.claude/ 目录结构，把每个组件接到正确位置。
我需要 ccprivate 吗？
只有当你需要存 API key、token 或私有配置时才需要。ccconfig 可独立运行 — 克隆、运行 setup、搞定。ccprivate 是给不想放进公开仓库的秘钥准备的。
可以只取部分用吗？
可以。一切都是可选的。Skills 按条安装 /plugin install。Rules 是独立文件 — 留你想要的，删掉不需要的。符号链接系统能干净地处理部分采用。
支持哪些操作系统？
Linux 和 WSL2（Windows）完整支持且每日测试。macOS 支持计划中 — 大部分脚本已可用，但 auto-sync 需 launchd 等价实现。欢迎 PR。
稳定吗？能用于生产吗？
ccconfig 由维护者在多台机器上每天使用。Release 分支只接收打过 tag 的测试版本。Auto-sync 已连续运行数月。话虽如此，这是个人基础设施项目 — 不是企业产品。读代码，都是 bash 脚本和符号链接。
