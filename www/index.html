What It Does

Infrastructure for your
Claude Code environment.

Not a tool you run. A foundation you clone. Everything is files. Everything is git. Everything is composable.

🔗

Smart Symlinks

Rules, skills, agents, MCP configs — all linked from one git repo to ~/.claude/. One setup-links.sh wires everything.

🔐

Public / Private Split

ccconfig is open source, safe to fork. Secrets live in ccprivate. Symlinks bridge the two. Zero tokens in public repo.

How It Works

Three steps.
Thirty seconds each.

1

Clone the repo

git clone --branch release https://github.com/mengfanchun2017/ccconfig.git

Release branch is stable. Main is development. You always get the tested version.

2

Run setup

bash ccconfig/setup-links.sh

Creates symlinks from ~/.claude/ to your repo. Rules, skills, agents, MCP configs, settings — all wired up instantly.

3

Let it sync

systemctl --user enable auto-sync

Background monitor watches for changes. Pushes to git automatically. Your other machines pull and stay current.

FAQ

You might wonder.

How is this different from dotfiles?
Dotfiles sync files. ccconfig manages versions, dependencies, auto-updates, and bridges public/private repos with symlinks. It's dotfiles with a build system — setup-links.sh understands the full ~/.claude/ directory structure and wires every component to the right place.
Do I need ccprivate?
Only if you have API keys, tokens, or private configs. ccconfig works standalone — clone, run setup, you're done. ccprivate is for secrets you don't want in a public repo.
Can I use only parts of it?
Yes. Everything is optional. Skills install per-line with /plugin install. Rules are individual files — keep the ones you want, delete the rest. The symlink system handles partial adoption cleanly.
What OS does it support?
Linux and WSL2 (Windows) are fully supported and tested daily. macOS support is planned — most scripts work already, but auto-sync needs a launchd equivalent. PRs welcome.
Is it stable? Can I trust it in production?
ccconfig is used daily across multiple machines by its maintainer. The release branch only receives tagged, tested versions. Auto-sync has been running continuously for months. That said, it's a personal infrastructure project — not an enterprise product. Read the code, it's all bash scripts and symlinks.
