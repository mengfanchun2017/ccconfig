# Security Policy

## Supported Versions

The latest commit on `main` branch.

## Reporting a Vulnerability

**Do not open a public issue.** Instead, report vulnerabilities privately:

1. GitHub: "Report a vulnerability" button on the Security tab
2. Or email: `mengfanchun1981@163.com`

You should receive a response within 72 hours. Include:

- Description of the vulnerability
- Steps to reproduce
- Affected versions/components
- Any potential fixes you've identified

## What to Expect

1. Acknowledgment within 72 hours
2. Status update within 7 days
3. Disclosure timeline agreed with reporter
4. Credit in CHANGELOG (unless you prefer anonymity)

## Scope

This policy covers the ccconfig repository and its runtime scripts. Third-party MCP servers and external dependencies should be reported to their respective maintainers.

## Safe Harbor

Security research conducted in good faith under this policy is:

- Exempt from DMCA/circumvention claims
- Exempt from ToS violations related to testing
- Protected from legal retaliation if conducted per policy guidelines

Thanks for helping keep ccconfig secure.
